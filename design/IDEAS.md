## 🦫 Loading Battle (гейміфікація завантажень) — Val дуже любить це, тримаємо як фірмову фішку

**Суть:** замість нудного прогресбара — під час КОЖНОГО завантаження (рендер, збірка, локалізація, транскрипція) йде **бій**: наш герой лупашиться з монстрами. Розважає юзера, поки він чекає.

**Механіка:**
- Старт завантаження → починається бій. Герой б'є монстра, монстр здригається, новий замах (петля).
- Коли монстр **майже готовий померти** — по ньому можна **клікнути**, і герой робить **СУПЕР-МУВ** (фінішер). Легка гейміфікація.
- Монстр гине → приходить **новий** монстр.
- **HP монстра прив'язане до обсягу роботи.** Приклад: 5 хуків через HeyGen = **5 хай-левел боссів** (кожна одиниця роботи = монстр/бос). Що більше/важче завдання — то «жирніший» бій.
- **Значок гучності** біля завантаження: натискаєш → чути **звуки бою + епічне музло** (стиль AUDIOMACHINE / трейлерний epic). За замовч. тихо.

**Стиль (важливо):**
- Герої — **стікери/паперові клаптики**, рухаються так, **наче маленький хлопчик грається вирізаними з паперу фігурками**, імітуючи рухи. **Стоп-моушн** вайб (рвана низькокадрова анімація).
- Герої — **гуманоїдні звіроподібні, мультяшні, чібі** (велика голова), у стилі **геройського фентезі**.
- Капібара = один з героїв. Зробити її «капібарнішою» (бочкоподібне тіло, тупа квадратна морда, маленькі вушка, куці лапи).

**Live-service:** періодично оновлюємо міні-фічу — **нові муви ударів, нові герої, нові монстри, нова музика**. Випадковий герой на кожне завантаження.

**Технічно:** асети векторні (SVG) + CSS-анімації, нічого зовнішнього не треба. (Опційно — PNG спрайт-шити, якщо захочемо конкретний арт.) Звук — Web Audio або короткі семпли.

**Статус:** ідея зафіксована, будуємо пізніше. Зараз працює простий бар із капібарою + таймер (правий нижній кут).

## 🌍 Авто/напів-авто локалізація (на паузі)

- Рекомендація: **напівавтоматика**. Авто вся механіка (дубляж/ліпсінг → перетранскрипція → успадкування EN-стилю → лягає готовим хуком), але **людина перевіряє текст** (маркетинг!).
- Рантайм-перемикач **checkpoints: None / Text-review / Final-OK**. Один рушій — три швидкості.
- **Лоток рев'ю**: інлайн-правка перекладу + зелена галочка / «прийняти все».
- Бонус: **глосарій «не перекладати / завжди писати так»** (рушій уже вміє `replacements`). + **fit-check** (німецька довша → не вилазить за рядок).

## 📦 Перед релізом (пакування для команди ~10 людей, маки M3)

- **№1 — запуск одним кліком** (`run.command` вже є — допиляти: авто-venv, залежності, відкрити браузер).
- **Залежності першого запуску**: ffmpeg, python-пакети, модель Whisper (качається при першій транскрипції).
- **Екран ключів/налаштувань**: ключі локально на машині (не в спільній папці), попередження про витрати (дубляж/HeyGen платні).
- **Дружні помилки** (без Python-трейсбеків): офлайн, рендер впав, кліп >32MB для HeyGen, нема ffmpeg.
- **Файли/чистка**: куди лягають фінали, де бібліотека, чистка тимчасових.
- **Оновлення**: як команда отримує нові версії + версія в UI.

## 📊 ROI / метрики (для C-level пітчу — ПРІОРИТЕТ)

Мета боса: показати **скільки людино-годин і грошей економимо**. План:
- Логувати кожну операцію (рендер, локалізація): час, тип, к-сть креативів, мови, тривалість (таймер уже є).
- Екран **Stats / ROI**: усього креативів, локалізацій, мовних версій; час в апці vs оцінка ручної/агенційної роботи.
- Формула: **год. зекономлено = креативи × (ручний_час − наш_час)**; **$ зекономлено = креативи × $20 (агенція)** + дубляж замість агенції.
- Базові константи (хв на креатив вручну, $/креатив) — налаштовувані.
- Експорт зведення (CSV/скрін) для презентації.

## 🔒 Безпека / ліцензування (чесно)

- **MAC-адреса = пароль**: це **детерент, не справжній замок** — апка локальна, код у людини, технічний юзер легко обійде (та й MAC спуфиться). Дає «поріг», не захист.
- Реальний актив — не код (ffmpeg + публічні API), а **API-ключі** (коштують грошей) і бренд-стилі/бібліотека. Фокус безпеки: **не тримати ключі у спільній папці**, а ввести локально / проксіювати.
- Прагматичний варіант: **allowlist machine-ID** (uuid.getnode()) при старті + можливість відкликати. Просто, відлякує шеринг.
- Справжній enterprise-захист: винести платні частини (dub/HeyGen) за **наш бекенд з авторизацією per-user** — тоді шерити фронт безсенсово. Важче, але надійно.
